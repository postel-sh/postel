# scripts/spec-drift-deferred.txt
#
# Requirements deferred to a later release. The companion script
# scripts/check-spec-drift.mjs exempts these from its drift check until
# they land tests in `typescript/packages/`. The file is auditable and
# grep-able; an entry naming a requirement that no longer exists in any
# `openspec/specs/<cap>/spec.md` is itself a drift failure.
#
# Lines starting with `#` are comments. Blank lines are ignored. Each
# remaining line is the verbatim title from a `### Requirement:` block.
#
# Reduction discipline: when a PR adds a test that covers a deferred
# requirement, REMOVE the corresponding line in the same PR.
#
# Out-of-scope categories for the v0.1.0 TS receiver track:
#   - sender / outbox / retry / fanout / replay / endpoint state machine
#   - storage adapters and host-transaction passthrough
#   - multi-tenancy, observability, filtering
#   - compliance suite internals (Track A owns the runner)
#   - distribution lifecycle policies (SemVer, deprecation, migrations)
#   - api-surface-typescript surface beyond the v0.1.0 receiver-side shape
#
# Per `openspec/specs/compliance/spec.md` v0.1.0 scope, the in-scope set
# is ~17 receiver / key-management / standard-webhooks-compliance items.
# Everything else is listed below.
