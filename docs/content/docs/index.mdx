Sending and receiving webhooks is easy. Doing it **reliably and securely** is hard — retries, replay, signature verification, key rotation, idempotency, raw-bytes preservation. That's where Postel comes in: a polyglot library that handles those for you. It runs inside the host application against the host application's existing relational database (Postgres, MySQL, SQLite, …), and ships in multiple languages over time. The TypeScript implementation ships first; Go, Python, and Rust follow.

> Svix is for when webhooks are your **product**.
> Postel is for when webhooks are a **feature** of your product.

## Status

Pre-alpha. The TypeScript receiver ships today through the `Postel({ inbound })` factory in `@postel/core`: multi-verifier composition, JWKS consumer, dedup helper, raw-bytes-preserving framework adapters. The sender — `postel.outbound.send`, the transactional outbox, retries, fanout, replay — lands in v0.2.0; its types ship today and the runtime throws `NotImplementedError` until then. Polyglot ports (Go, Python, Rust) follow.

We document the spec ahead of the implementation. Pages covering unimplemented capabilities are explicit about it — assume nothing ships until the [API reference](/docs/api) lists it.
