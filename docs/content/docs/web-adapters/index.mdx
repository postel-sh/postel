Every web adapter is a thin **gate** over [`@postel/http`](/docs/reference/http): it reads the raw request bytes, runs the verifier(s) you configured on a source, maps any `PostelError` to the right HTTP status, optionally dedup-acks, and only then hands control to your handler. You keep writing a normal handler — you don't re-implement verify → error-mapping → ack in every route.

The error→status policy, byte preservation, and the dedup-ack signal live **once** in `@postel/http`; each adapter binds them to its framework's idiom (middleware, preHandler, guard), so they can't drift. A non-`PostelError` (e.g. a programming bug) bubbles as 5xx — never miscast as a 4xx.

## The shape every adapter shares

- **`verifyWebhook(source, opts?)`** — the framework-native gate (Hono/Express middleware, Fastify preHandler, NestJS guard). Your handler runs after it, with the verified result on the request context.
- **`withWebhook(source, handler, opts?)`** — folds the gate and a single handler into one.
- **`xAdapter(postel)`** — `.verify(key)` / `.guard(key, handler)` bound by *source key* (type-checked against your config), plus `.jwks(provider)` to publish keys.

## Status

| Framework / runtime | Package | Ships? | Form |
|---|---|---|---|
| Hono | `@postel/hono` | ✅ | Middleware gate + adapter object |
| Express | `@postel/express` | ✅ | `express.raw()` + middleware gate |
| Fastify | `@postel/fastify` | ✅ | Plugin + preHandler gate |
| NestJS | `@postel/nestjs` | ✅ | Module + `WebhookGuard` + decorators |
| Any Fetch runtime (Deno, custom, …) | [`@postel/http`](/docs/reference/http) | ✅ | `fetchWebhook(source)` |
| Next.js Route Handlers | `@postel/nextjs` | ⏳ stub | Use `@postel/http` meanwhile |
| Bun.serve | `@postel/bun` | ⏳ stub | Use `@postel/http` meanwhile |

## Both directions

Adapters also mount the **outbound** HTTP surface: `.jwks(() => postel.outbound.keys.publicJwks())` publishes your current public keys (see [key rotation](/docs/inbound/key-rotation)), and the [`@postel/admin`](/docs/outbound/admin) control plane mounts through each adapter (natively on Hono; via `fetchToExpress` / `fetchToFastify` on Node).
