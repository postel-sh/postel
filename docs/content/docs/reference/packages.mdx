Postel is a workspace of focused packages. Pick the ones you need; the rest don't get installed.

## Available today

| Package | Used for | Notes |
|---|---|---|
| **`@postel/core`** | [Inbound](/docs/inbound) and [Outbound](/docs/outbound) — the `Postel({ inbound, outbound })` factory and primitives | Zero runtime dependencies. The outbound runtime is available, including the in-process `InMemoryStorage` adapter (`import { InMemoryStorage } from "@postel/core"`). |
| **`@postel/http`** | [Framework-agnostic webhook HTTP core](/docs/reference/http) | The verification gate (`handleInbound`, `fetchWebhook`) and the shared `PostelError`→HTTP-status policy every framework adapter binds to. Depends only on `@postel/core`. |
| **`@postel/hono`** | [Hono](/docs/web-adapters/hono) framework adapter | `verifyWebhook()` / `honoAdapter(postel).verify()` gate middleware + `withWebhook()` wrapper. `honoVerify`/`postelHono` kept as deprecated aliases. |
| **`@postel/express`** | [Express](/docs/web-adapters/express) framework adapter | `verifyWebhook()` (mounts `express.raw()` + the gate) / `expressAdapter(postel).verify()`; sets `req.postel`. |
| **`@postel/fastify`** | [Fastify](/docs/web-adapters/fastify) framework adapter | `fastifyPostel` raw-body plugin + `verifyWebhook()` preHandler / `fastifyAdapter(postel).verify()`; sets `req.postel`. |
| **`@postel/nestjs`** | [NestJS](/docs/web-adapters/nestjs) framework adapter | `PostelModule.forRoot(postel)` + `WebhookGuard(key)` guard + `@Event()` / `@WebhookResult()` decorators. |
| **`@postel/admin`** | [Admin control plane](/docs/outbound/admin) over the outbound sender | `adminRouter(postel, { authorize })` Fetch router (endpoint CRUD, replay, keys) with default-deny auth + tenant scoping; mount via Hono / `fetchToExpress` / `fetchToFastify`. |
| **`@postel/sqlite`** | [SQLite storage adapter](/docs/storage/sqlite) | Standalone — Postel owns the SQLite database (`postelSqlite`). Full outbound `Storage`; also exports `sqliteDedupAdapter` for inbound dedup. |
| **`@postel/pg`** | [Postgres storage adapter](/docs/storage/pg) | Standalone — `postelPg({ connectionString })`. Full outbound `Storage` (`FOR UPDATE SKIP LOCKED`, `LISTEN`/`NOTIFY`); also exports `pgDedupAdapter` for inbound dedup. |
| **`@postel/storage-helpers`** | Zero-DB helpers for [custom adapters](/docs/storage/custom-adapters) | Dialect codecs, capability flags, row encode/decode, migration SQL, callback registry — what every adapter reuses. |

## Stubs today (name reserved, no runtime yet)

| Package | Will be | Status |
|---|---|---|
| `@postel/nextjs` | Next.js Route Handler ergonomics | Planned |
| `@postel/bun` | Bun.serve adapter | Planned |

Each "stub" package exports only `__postelPackage` today. Don't depend on them in production yet — the framework-agnostic [factory recipe](/docs/inbound/verify) works directly against each runtime's native `Request`.

## Storage adapters

The `postel.outbound.*` runtime runs against any `Storage` adapter: the in-process `InMemoryStorage` (in `@postel/core`), the shipping standalone adapters [`@postel/pg`](/docs/storage/pg) and [`@postel/sqlite`](/docs/storage/sqlite), or — landing next — the query-builder/ORM adapters:

| Package | Will be |
|---|---|
| `@postel/kysely` | Kysely query-builder storage adapter |
| `@postel/drizzle` | Drizzle ORM storage adapter |
| `@postel/prisma` | Prisma storage adapter |

The storage adapter matrix is per [ADR 0007](https://github.com/postel-sh/postel/blob/main/decisions/0007-storage-strategy.md) — Postgres / SQLite as the canonical first-class databases, ORM-specific adapters in their own packages. The external worker strategies (`BullMQ`, `PgBoss`) are likewise config slots today and throw `NotImplementedError` until their adapters ship; the in-process worker pool is the supported runtime.

## Compliance & tooling

| Package | What it is | Status |
|---|---|---|
| `@postel/compliance` | The executable behavioral oracle. Vendor-neutral test suite for any Standard Webhooks producer/receiver. A port version `X.Y.Z` claims conformance by passing `@postel/compliance@X.Y.*`. | Receiver + sender; the sender corpus executes end-to-end against `@postel/compliance-driver`. |
| `@postel/compliance-driver` | The HTTP control-plane the compliance runner drives in sender mode — it wraps a real `Postel` outbound instance so the cross-port runner can reset state, register endpoints, send, advance the clock, and observe outgoing deliveries. | Workspace-internal harness for the suite; not a production dependency. |
| `@postel/test` | Workspace-internal test fixtures (signed payload generators, vector builders). | Internal — not published. |
